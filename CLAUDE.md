# sabotage(サボタージュ)— 効率よくだらだらディズニー執事

東京ディズニーリゾートを「効率よくだらだら」過ごすためのAI執事。
sabotageの原義は「組織的・計画的な怠業」— 無計画にだらけるのではなく、
戦略として怠ける。思想的基盤は荘子『逍遥遊』(目的のない遊びこそ最高の
自由)であり、それを計画的怠業という戦術で実装する。**無為の外注。**

思想対応表:
- 庖丁解牛 → 人流ナウキャスティング(群衆の筋目を読み、隙間だけを通る。刃=体力を消耗させない)
- 朝三暮四 → ピークエンドの法則(総量ではなく配列が満足を決める)
- 木鶏 → 障害時の平常心(スプラッシュ停止で騒がない)
- 鯤 → セレンディピティ枠(計画が偶然の風に乗って鵬に化ける瞬間のための余白)

## 絶対原則(目的関数)
- 「乗った数」を最大化しない。最適化対象は「最高の一瞬 × 最高の締め」の確保と、
  それ以外の時間の着席・飲食時間の最大化。
- 立ち待ちは損失。移動距離は損失。当日の判断コストも損失(ユーザーに考えさせない)。

## 心理設計原則(5条)
1. **自壊する信号の原則**: アプリの待ち時間表示は号令であり、魅力的な数字ほど速く死ぬ。
   意思決定と表示には常に「群衆補正込みの到着時予測値」を使う。遠方目標の生の現在値を
   ユーザーに見せることを禁ずる(表示層でのデバイアス)。
2. **同点stayの原則**: 人間には行動バイアスがある(動いて失敗より何もせず失敗を嫌う)。
   決断関数は同点を常にstayへ倒す(src/sabotage/mind/decision.py)。
3. **ユリシーズ契約**: 移動を提案する時は、出発前に撤退条件(着時◯分超なら無条件離脱)を
   契約させる。到着後の再交渉は認めない。撤退の後悔は執事が引き受ける。
4. **level-2の原則**: 園内の大半はアプリに追随するlevel-1プレイヤー。執事はその反応を
   一手先読みするlevel-2として振る舞う。シグナルへの素直な追随は設計上の敗北。
5. **網の原則**: シグナルは追わず、シグナルが発生しやすい場所の近くに先に座らせる
   (コロケーション)。stayは受け身ではなく待ち伏せである。

## アーキテクチャ(4層)
1. **data**: 非公式APIを5分間隔でポーリング → SQLiteに蓄積(生JSONも保存)
2. **analysis**: 人流ナウキャスティング(衝撃波・雨の再配分・フェアバリュー・群衆弾性)
3. **planning**: 行程国会 — マルチエージェント討論で「順序」ではなく「ポリシー」を生成
4. **execution**: BCPポリシーエンジン + 当日執事ループ(指示は「動け/座れ/食え」の3種のみ)

## 技術規約
- Python 3.11+ / httpx / SQLite / pytest。パッケージ名は sabotage。
- フレームワークは最小限、動くものを最速で。過剰な抽象化禁止。
- フェーズをまたぐ先回り実装禁止。
- 欠測・遅延・API仕様変更を前提に設計する(欠測はFETCH_FAILED行として記録する)。
- 外部APIは必ず実レスポンスを確認してからスキーマを確定する。テストは実形状の
  フィクスチャで回し、ネットワークに依存しない。

## データ倫理・制約(厳守)
- 非公式アグリゲータへの礼儀: ポーリングは5分間隔以上+ジッター、User-Agent明示。
- Queue-Times使用時は「Powered by Queue-Times.com」表記とリンクを必ず付ける。
- 東京ディズニーリゾート公式アプリ/APIのリバースエンジニアリングは行わない。
- 取得データは私的利用の範囲に留める。
